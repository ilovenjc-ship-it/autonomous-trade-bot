# Contemplations — Project Ari

> Self-audit / framework-application artifacts. Where the **Library** shelf
> (`MemoryBank/Library/`) captures *outside-in* reads (we read what someone
> else wrote, decide what to lift / counter / validate / skip), the
> **Contemplations** shelf captures *inside-out* reads — we apply an external
> framework, doctrine, or comparison to Project Ari's own architecture and
> record what falls out.

## When to file here vs in Library

- **Library (`MemoryBank/Library/`)** — the artifact is *"I read X and here is
  what to lift / counter / validate / skip."* Protocol: header / why-it-matters
  / Lifts / Counterfactuals / Validations / Skip / Vocabulary.
- **Contemplations (this shelf)** — the artifact is *"X is a framework or lens;
  let's apply it to Project Ari and see what falls out."* Protocol: framing
  paragraph / framework's own structure (binary filters, scoring rubric,
  decision tree) walked filter-by-filter / honest synthesis at the end /
  what-this-does-NOT-do guard / cross-references.

A piece can graduate from Contemplation to Library if it triggers a Library
read of the underlying source. A piece can graduate from Contemplation to
STATE.md inscription if a vocabulary term or decision-log entry warrants
elevation — descriptive inscriptions autonomous per D-23, prescriptive
inscriptions still require operator green-light.

## Inscription authority

Per **D-23 (Session XLIV Day 14, 2026-05-27)**: descriptive contemplations may
be filed autonomously when the trigger is named, the framework is cited, and
the rationale is filed alongside the analysis. Prescriptive recommendations
emerging from a contemplation still require operator green-light before
inscription to STATE.md §3 (vocabulary) or §4 (decision log).

Same boundary as Library Night: descriptive = autonomous; prescriptive =
operator-greenlit.
